# FonsClarus Engineering Blog Theme - Plan

## Overview

A minimal, fast, production-ready Hugo theme for a technical engineering blog focused on embedded systems and software engineering content. No JS frameworks, no visual clutter, clean typography.

## Theme Name: `fonsclarus`

## Architecture & Design Decisions

### 1. Base Template (`baseof.html`)
- HTML5 doctype, `<html lang="...">` set from Hugo config
- Block structure: `head`, `header`, `main`, `footer`
- Main content block for child templates to override
- TOC (table of contents) block — auto-generated if page has headings
- Dark mode CSS class toggle on `<html>` element

### 2. Homepage (`index.html`)
- Site title + tagline
- Links to Blog, Projects, About sections
- Minimal hero area — just text, no images or animations

### 3. Single Article Page (`single.html`)
- Title (h1), date, optional tags (linked to taxonomy pages)
- Auto-generated table of contents from headings (via Hugo `.TableOfContents`)
- Content rendered via `.Content`
- Tag list at bottom

### 4. List Page (`list.html`)
- Section title (e.g., "Blog", "Projects")
- Simple list of posts: title (linked), date, optional summary/description
- No cards, no thumbnails, no pagination fluff

### 5. Partials

#### `head.html`
- Meta charset, viewport
- `<title>` and `<meta name="description">` from page front matter or site config
- Open Graph basic meta tags (og:title, og:description, og:type)
- Canonical URL
- CSS link (Hugo Pipes with `toCSS` and `minify`)
- No external resources (fonts, CDNs, analytics)

#### `header.html`
- Site logo/name (text link to homepage)
- Navigation: Home, Blog (/blog), Projects (/projects), About (/about)
- Dark mode toggle button (sun/moon icon using unicode characters)
- Uses `.Site.Menus` for flexibility, with fallback hardcoded items

#### `footer.html`
- Copyright line with current year
- Powered by Hugo text
- Minimal — 1-2 lines max

### 6. CSS (`main.css`)
- **Reset**: Minimal box-sizing, margin/padding reset
- **Typography**: System font stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, sans-serif`), 16px base, 1.7 line-height for readability
- **Layout**: Centered column max-width 800px, 1.5rem horizontal padding on mobile
- **Navigation**: Horizontal inline list, subtle active state
- **Content**: Proper heading hierarchy (h1-h6), paragraph spacing, list styling, blockquotes, horizontal rules
- **Code**: `--mono-font` stack (`'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, monospace`), `#f5f5f5` background, `#e8e8e8` border, `overflow-x: auto`, border-radius
- **Inline code**: Slight background tint, monospace
- **Table of Contents**: Left sidebar on wide screens or collapsible on mobile, clean nested list
- **Dark mode**: `prefers-color-scheme: dark` media query, CSS custom properties for colors
- **Tags**: Inline flex list, subtle styling
- **Images**: `max-width: 100%`, `height: auto`
- **Links**: Underline on hover, distinct color
- **Copy button**: Positioned absolutely in code block wrapper, appears on hover
- **No animations, no transitions, no shadows, no gradients**

### 7. Code Highlighting
- Use Hugo's built-in `chroma` highlighter (configured via `hugo.toml`)
- CSS classes generated by Hugo highlight shortcode
- Dark mode aware via CSS custom properties on `.highlight` class

### 8. SEO
- Each page gets unique `<title>` and `<meta name="description">`
- Open Graph tags in `head.html`
- Clean semantic HTML5 structure (`<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`)
- Proper heading hierarchy

### 9. Dark Mode
- CSS custom properties for colors: `--bg`, `--text`, `--accent`, `--code-bg`, `--border`
- Light mode as default (no media query needed for light values)
- `@media (prefers-color-scheme: dark)` overrides custom properties
- Manual toggle via checkbox in header — adds `[data-theme="dark"]` to `<html>`
- Toggle state persisted? No — keep it simple, no localStorage

### 10. Copy Button for Code Blocks
- Tiny vanilla JS snippet (~15 lines, no dependencies)
- Appended to each `<code>` block wrapper after page load
- Button text: "Copy" — changes to "Copied!" on click (2s timeout)
- Pure CSS positioning, no extra libraries

## `hugo.toml` Updates Needed (in root site, not theme)

```toml
baseURL = 'https://example.org/'
locale = 'en-us'
title = 'FonsClarus Engineering'

theme = 'fonsclarus'

# Code highlighting
[markup]
  [markup.highlight]
    codeFences = true
    guessSyntax = false
    lineNos = false
    noClasses = true
    tabWidth = 4

# Taxonomies
[taxonomies]
  tag = 'tags'
  category = 'categories'

# Menu
[[menu.main]]
  name = 'Home'
  url = '/'
  weight = 1
[[menu.main]]
  name = 'Blog'
  url = '/blog'
  weight = 2
[[menu.main]]
  name = 'Projects'
  url = '/projects'
  weight = 3
[[menu.main]]
  name = 'About'
  url = '/about'
  weight = 4

# Pagination
[params]
  description = 'Embedded systems and software engineering blog'
```
